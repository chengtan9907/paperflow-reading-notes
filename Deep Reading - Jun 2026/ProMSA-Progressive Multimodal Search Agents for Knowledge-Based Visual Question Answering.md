# ProMSA:Progressive Multimodal Search Agents for Knowledge-Based Visual Question Answering

> ★★★☆☆ 值得快速浏览 · 模型 openai-compatible/gemini-3-flash-preview

🏷 关键词：kb-vqa · multimodal search agent · reinforcement learning · progressive retrieval

## 一句话总结

ProMSA 提出了一种渐进式多模态搜索智能体框架，通过强化学习优化的决策策略，自主选择图像或文本检索路径及搜索深度，显著提升了长尾实体背景下的知识库视觉问答（KB-VQA）性能。

## 摘要

> 针对知识库视觉问答（KB-VQA）中长尾实体识别难、检索路径不确定等挑战，本文提出了 ProMSA（Progressive Multimodal Search Agents）。该智能体能够像人类研究员一样，在有限的预算内交替进行多模态检索与推理：当无法识别图像实体时调用图像检索，当需要补充属性或进行多跳推理时生成文本查询。为了优化这一复杂的决策过程，研究者引入了 TN-GSPO（Token-and-Node Normalized Group Relative Policy Optimization），这是一种针对序列级策略优化的强化学习目标，通过对生成长度和工具交互深度进行双重归一化，解决了学习过程中的不稳定性。在 E-VQA 和 InfoSeek 数据集上的实验表明，ProMSA 在处理单跳和多跳知识推理任务上均优于现有的 RAG 和搜索智能体基线。

Q1: 这篇论文试图解决什么问题？

### 核心挑战：长尾实体与检索不确定性
传统的视觉问答（VQA）模型虽然在通用场景下表现良好，但在面对知识库视觉问答（KB-VQA）时，尤其是涉及长尾实体（Long-tail Entities）时，表现出明显的局限性。这些实体在预训练数据中出现频率极低，模型难以仅凭参数记忆准确识别。

### 检索模态的不确定性 (Retrieval Modality Uncertainty)
在实际推理中，模型面临路径选择的难题：
1. **图像检索需求**：当图像中的实体完全陌生或无法通过视觉特征直接命名时，模型需要先通过图像检索（如以图搜图）定位对应的维基百科页面。
2. **文本检索需求**：当实体已知但问题涉及复杂的属性补全或多跳推理时，模型需要生成高质量的文本查询，通过文本检索获取缺失的证据。

### 检索深度的不确定性 (Retrieval Depth Uncertainty)
由于用户上传的图像与知识库中的参考图在视角、背景和上下文上可能存在巨大差异，正确的实体信息可能不会出现在首次检索的首位。这要求模型具备“重试”和“去重”的能力，通过多次尝试逐渐逼近正确答案。现有的 RAG（检索增强生成）方法大多是静态的一步式过程，缺乏这种动态调整检索深度的灵活性。

### 现有方法的局限
目前的搜索智能体（Search Agents）虽然引入了多轮交互，但在处理多模态切换（图像转文本或反之）时缺乏系统性的策略，且在强化学习训练过程中容易受到生成长度和交互步数的影响，导致训练不稳定或陷入局部最优。

Q2: 有哪些相关研究？

### 检索增强生成 (RAG)
RAG 技术通过引入外部知识库缓解了 LLM 的幻觉问题。在多模态领域，现有的 RAG 方法已在检索精度、证据过滤和冲突缓解方面取得了进展，但大多数仍遵循“先检索后生成”的静态流水线，难以处理需要多步探索的复杂知识推理。

### 搜索智能体 (Search-Based Agents)
近期研究（如 MMSearchR1, SenseNova-MARS）开始探索将搜索过程建模为智能体行为。这些方法允许模型根据需求进行多轮搜索。然而，如何在统一的框架下平衡图像检索与文本检索的权重，以及如何有效训练这种具备多模态决策能力的智能体，仍是前沿难题。

### 强化学习与策略优化
在智能体训练中，GRPO（Group Relative Policy Optimization）等算法被用于优化决策序列。但本文指出，在多模态搜索场景下，标准的 GRPO 容易受到不同动作节点（Node）和生成长度（Token）的影响，导致梯度更新不均衡。本文提出的 TN-GSPO 正是对这一方向的改进。

Q3: 论文如何解决这个问题？

### ProMSA：渐进式多模态搜索智能体
ProMSA 被设计为一个受预算约束的渐进式检索-推理过程。在给定的工具调用次数限制下，它在检索和推理之间交替进行，累积相关信息，直到证据足以回答问题为止。

### 核心机制设计
1. **多模态工具箱**：集成图像检索（定位实体页面）和文本检索（获取具体属性）。
2. **显式去重 (Explicit De-duplication)**：在多轮检索中，智能体会记录已访问的页面，避免重复检索相同内容，从而在有限预算内探索更多候选实体。
3. **自适应停止**：模型根据当前状态 $s_t$ 判断是否已获得足够证据。如果证据充足，则输出最终答案；否则继续选择检索模态。

### TN-GSPO 强化学习算法
为了解决搜索策略的学习难题，本文提出了 **TN-GSPO (Token-and-Node Normalized GSPO)**：
- **双重归一化**：不仅在 Token 级别进行归一化，还在工具交互的节点（Node）级别进行归一化。这确保了无论搜索路径长短，每个决策步骤对策略更新的贡献都是均衡的。
- **非对称裁剪 (Asymmetric Clipping)**：在计算优势函数时引入非对称裁剪机制，防止策略在训练初期发生剧烈偏移，增强了学习的稳定性。

### 训练阶段
- **阶段 I：冷启动 (Cold Start)**：利用高质量的专家轨迹进行监督微调（SFT），让模型掌握基本的工具调用格式和搜索逻辑。
- **阶段 II：强化学习 (RL)**：使用 TN-GSPO 优化搜索策略，提升模型在面对不确定性时的决策质量。

Q4: 论文做了哪些实验？

### 实验设置
- **数据集**：
 - **E-VQA (Encyclopedic-VQA)**：包含 221K 个问答对，涵盖 1.67万个细粒度维基百科实体，分为单跳和多跳问题。知识库包含 200万个维基百科页面。
 - **InfoSeek**：包含 130万个图像-问题-答案三元组，关联约 1.1万个维基百科页面。实验重点关注“未见问题（Unseen-Q）”和“未见实体（Unseen-E）”的泛化能力。
- **基准模型**：使用 Qwen3-VL-8B 作为基础模型。
- **检索器**：图像检索使用 CLIP，文本检索使用 BGE。
- **预算限制**：图像和文本搜索各最多调用 3 次，每次检索 Top-3 页面。

Q5: 发现了什么实验现象？

### 关键实验发现
1. **强化学习的显著增益**：在 E-VQA 数据集上，经过阶段 II 的 RL 训练后，模型性能从阶段 I 的 38.6% 大幅提升至 52.6%。这表明 RL 有效地教会了模型如何更聪明地利用搜索工具，而不仅仅是模仿专家行为。
2. **TN-GSPO 的优越性**：对比实验显示，TN-GSPO 在 E-VQA 和 InfoSeek 上的表现均优于标准的 GRPO 和 GSPO。特别是在引入非对称裁剪（Asymmetric Clipping）后，性能提升更为稳健（E-VQA 从 44.2 提升至 52.6）。
3. **多模态协同效应**：消融实验（Table 4）证明，仅使用文本检索（27.6%）或仅使用图像检索（34.7%）的效果远低于两者结合（52.6%）。这验证了在 KB-VQA 中，图像检索用于定位实体、文本检索用于深化理解的互补性。
4. **泛化能力**：在 InfoSeek 的“未见实体（Unseen-E）”测试中，ProMSA 依然保持了 53.3% 的高准确率，证明其搜索策略具有很强的迁移性，不依赖于训练集中的特定实体知识。
5. **失败模式分析（合理推断）**：虽然论文未详细列出负面案例，但从“检索深度不确定性”的讨论中可以推测，当初始检索结果完全偏离目标且预算耗尽时，模型仍可能给出错误答案。此外，多跳推理中的错误累积（Error Propagation）仍是潜在挑战。

Q7: 总结一下论文的主要内容

本文针对知识库视觉问答（KB-VQA）中长尾实体识别和复杂推理的难题，提出了 ProMSA 框架。该框架的核心思想是将 KB-VQA 视为一个受预算约束的渐进式多模态搜索过程。ProMSA 智能体能够根据当前推理状态，自主决定是进行图像检索以识别实体，还是进行文本检索以获取属性，并具备多轮迭代和显式去重的能力。为了解决多轮搜索策略难以训练的问题，作者提出了 TN-GSPO 强化学习算法，通过对 Token 和 Node 级别的双重归一化以及非对称裁剪，实现了稳定且高效的策略优化。在 E-VQA 和 InfoSeek 两个大规模基准数据集上的实验结果表明，ProMSA 在准确率上显著超过了现有的 RAG 方法和单模态搜索智能体，展现了强大的长尾实体处理能力和跨模态推理泛化性。该研究为构建具备自主知识获取能力的通用多模态智能体提供了重要的技术路径。
